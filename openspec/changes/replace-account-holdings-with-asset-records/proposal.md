## Why

当前 `asset_account + asset_info + asset_holding` 模型要求资产先归属账户，再通过用户级资产主数据关联持仓；同时 `asset_info` 又被用作基金和股票自选关系。该结构不适合现金、银行卡、虚拟余额、基金、股票、债权和负债统一按金额记录的场景，也无法让用户先记“未细分基金/股票金额”，再逐步拆成具体标的。

本需求来源于已确认的资产目录和录入流程讨论，不直接关联现有 `docs/requirements/**/prd-*.md`。当前数据允许全部重建，因此本变更以新模型一次性替换旧模型，不设计存量迁移。

## What Changes

- 新增两级资产目录，区分系统目录与用户私有目录，并通过金额方向参与净资产计算。
- 以 `asset_record` 统一保存普通金额、未细分基金/股票金额和具体基金/股票持仓，删除账户和中间资产主数据。
- 支持从一条未细分金额原子拆出一条具体持仓，保持同目录、同币种金额守恒。
- 以只追加的 `asset_record_change` 保存金额、拆分和生命周期变更。
- 以独立 `watchlist_item` 保存用户基金/股票自选关系，替代 `asset_info`。
- 新增通用结构的最新汇率表；当前只维护外币兑人民币汇率，并支持按查询指定币种汇总资产。
- 重建开发数据和相关查询，不迁移现有账户、资产、持仓或变更历史。

## Capabilities

### New Capabilities

- `asset-catalog-records`：资产目录、统一资产记录、未细分金额拆分和变更历史。
- `watchlist-relations`：用户与公共基金/股票之间的独立自选关系。
- `exchange-rate-summary`：当前汇率维护和多币种资产汇总。

## Impact

- 影响数据库初始化脚本、迁移脚本和开发 Mock 数据。
- 影响 portfolio 领域模型、Case、Repository、DAO、Mapper XML、API 和 Trigger。
- 影响现有自选列表、批量加入、搜索自选状态和基金刷新目标选择。
- 需要更新根目录客户端契约；外部 `assetRef` 继续作为不透明 API 引用，内部映射改为 `asset_kind + asset_id`。
- 不改变 `fund`、`stock_market` 作为公共基金和股票当前事实的边界。
- 本次不实现具体标的分析、分析覆盖率、数量成本、自动汇率刷新、资产跨目录移动或历史数据迁移。
