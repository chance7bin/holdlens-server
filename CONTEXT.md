# HoldLens Server 领域语言

HoldLens Server 维护用户资产记录、自选关系以及基金和股票公共数据等长期业务事实。本词汇表用于统一这些事实在需求、设计和代码中的业务含义。

## 资产记录

**资产目录**：
用于分组和汇总资产记录的层级分类；目录通过 ID 保持身份，并表达父子关系、金额方向、展示顺序和启停状态，其中系统默认目录另有稳定业务编码。
_避免使用_：账户类型、标的类型

**系统默认目录**：
由系统预置且具有不可变稳定编码的资产目录。基金和股票默认目录分别触发对应的专用录入流程，但目录本身不保存标的类型。
_避免使用_：可删除目录、按名称识别的目录

**系统目录**：
对所有用户可见且不归属于任何用户的只读资产目录，由系统版本统一维护，并通过全局唯一的稳定编码保持身份。
_避免使用_：用户目录、管理员个人目录

**用户目录**：
由单个用户创建和维护、仅该用户可见和使用的资产目录。用户目录只以目录 ID 标识，不设置系统业务编码；它可以构建自己的层级或挂在系统分组目录下，但不能挂在系统叶子目录下。
_避免使用_：系统目录、共享目录

**删除用户目录**：
将没有子目录和活跃资产记录的用户目录标记为不可继续使用，同时保留目录及其历史引用；用户目录不物理删除。
_避免使用_：物理删除、删除系统目录

**金额方向**：
资产目录对净资产的影响方式；加项目录增加净资产，减项目录减少净资产。同一目录分支中的父子目录必须保持相同方向。
_避免使用_：正负金额、记录类型

**叶子目录**：
没有子目录的启用资产目录，是唯一可以创建新资产记录的目录；非叶子目录只负责分组和汇总。
_避免使用_：一级目录、固定二级目录

**资产记录**：
用户在资产目录下维护的一笔金额记录，可以表示资产、债权或负债；记录可以只包含金额，也可以关联具体基金或股票。资产记录创建后，其所属目录和公共标的身份不可改变。
_避免使用_：账户、持仓（作为全部资产记录的统称）

**普通金额记录**：
不关联具体基金或股票，也不用于继续拆分具体标的的资产记录，例如现金、银行卡余额、债权和负债。
_避免使用_：未细分金额、持仓

**记录金额**：
用户确认并保存的资产记录当前原币金额，是该币种下的权威事实。公共行情、数量、成本或汇率信息不得自动覆盖记录金额。
_避免使用_：实时估值、行情计算金额

**记录名称**：
资产记录创建时保存的名称。具体持仓的名称由服务端从公共基金或股票复制为只读快照，公共名称后续变化不自动覆盖该快照；普通金额和服务端生成的未细分金额名称可以由用户修改。
_避免使用_：公共标的当前名称、备注

**原币金额**：
资产记录按照创建时确定且不可修改的币种保存的金额；汇率换算只产生汇总结果，不改写原币金额或记录币种。
_避免使用_：本位币金额、换算后金额

**汇率**：
独立于用户和资产记录的当前币种换算关系，用于把原币金额转换为指定汇总币种。汇率结构支持通用币种对，但当前业务只保存“外币作为基准币、人民币作为报价币”的最新关系；汇率变化不得改写资产记录。
_避免使用_：用户汇率、资产记录金额

**汇总币种**：
一次资产汇总查询指定的目标展示币种；未指定时默认人民币。汇总币种不作为用户属性持久化。
_避免使用_：用户本位币、资产记录币种

**部分汇总**：
部分资产记录因缺少所需的外币兑人民币汇率而无法换算时，对其余记录生成的汇总结果。无法换算的记录必须被明确列出，不能按零金额处理。
_避免使用_：完整汇总、零金额资产

**持仓**：
关联了具体基金或股票的资产记录。现金、银行卡余额、未细分投资金额、债权和负债不称为持仓。
_避免使用_：账户、普通金额记录

**同标的聚合**：
在按标的展示或汇总时，临时合并同一用户关联到同一公共基金或股票的多条持仓；聚合不改变各资产记录的独立性。
_避免使用_：合并资产记录、账户持仓

**基金/股票录入模块**：
由基金或股票系统默认目录的稳定编码触发的固定业务流程，分别确定资产记录的基金或股票类型，并允许只记未细分金额或关联具体公共标的。该流程创建的记录始终归属其创建目录，不能改归其他目录。
_避免使用_：按目录名称识别的录入流程

**未细分金额**：
一条已经确定为基金或股票、但尚未拆分到具体标的的独立投资金额记录；同一资产目录可以存在多条。它参与当前资产汇总；具体标的分析由后续独立模块处理。
_避免使用_：分类总额、持仓

**拆分现有金额**：
将未细分金额的一部分转为同一资产目录、相同币种的一条具体持仓，操作前后对应资产目录在该币种下的总金额保持不变。多个具体标的通过多次拆分完成。
_避免使用_：新增资产

**新增资产**：
创建新的资产记录并增加对应资产目录的总金额。
_避免使用_：拆分现有金额

**归档资产记录**：
资产记录暂不计入当前资产，但仍保留其历史事实，并可以恢复为活跃状态。金额归零不会自动归档；未细分金额被全部拆分后归档原记录。
_避免使用_：关闭资产记录、删除资产记录、活跃零金额记录

**删除资产记录**：
将误录的资产记录标记为删除并保留其审计历史；删除不用于表达余额归零或暂不展示。
_避免使用_：物理删除、归档资产记录

**资产记录变更**：
对资产记录金额或生命周期产生影响的一次不可变、可审计事实；同一次拆分产生的源记录减少和目标记录增加属于同一个变更操作。纠错通过新增补偿变更表达，不修改既有历史。
_避免使用_：覆盖记录、无来源金额调整

## 自选

**自选标的**：
用户关注但不代表实际持有的公共基金或股票。自选标的与资产记录相互独立。
_避免使用_：自选资产、持仓

## 公共基金与市场数据

**基金目录**：
HoldLens 全局已知的公开基金身份集合，包含基金代码、简称、类型和拼音检索信息，用于基金发现和详情查询；进入基金目录不代表用户持有、自选，也不代表系统需要定期执行单基金刷新。
_避免使用_：持仓、自选标的、刷新目标

**基金目录同步**：
以外部目录本次成功返回的基金为增量覆盖范围；未返回的已有基金继续保留，不能仅因单次缺失而删除或停用。
_避免使用_：全表替换、缺失即删除

**刷新目标**：
进入公开基金详情或股票行情主动刷新范围的代码目标，通常由用户持有、自选、近期访问或系统任务注册产生；完整基金目录默认不等于刷新目标。
_避免使用_：持仓、基金目录

**近期查看基金**：
用户最后一次查看详情后的九十天内仍具有近期关注度的基金；它临时进入定时刷新范围，再次查看会延长窗口，过期后退出但不删除已有公开数据。
_避免使用_：自选标的、永久刷新目标

**基金详情异步回源**：
基金详情查询优先返回 server 已保存的公开数据；数据缺失或过期时创建后台刷新，查询请求不等待外部基金接口完成。
_避免使用_：查询时同步抓取、绕过 server 直连数据源

**基金收益快照**：
单只基金在最新数据日期下的当前净值与阶段收益事实；当前阶段只维护最新值，不表达每日历史序列。
_避免使用_：基金收益历史、净值走势

**基金收益覆盖状态**：
基金类型是否属于当前收益数据源的支持范围；数据源未覆盖表示当前阶段没有该类收益能力，不属于刷新失败。
_避免使用_：刷新失败、字段缺失

**基金当前前十大重仓**：
单只基金在最新公开披露季度中按排名选取的前十只股票持仓；当前阶段不表达更早季度、完整股票持仓或债券持仓。
_避免使用_：基金完整持仓、基金持仓历史

**数据日期**：
公开数据源声明的数据所属日期或报告期，例如收益日期或重仓披露季度；它表达数据内容截至何时。
_避免使用_：抓取时间、数据库更新时间

**抓取时间**：
系统实际完成某个基金数据切片获取的时间；目录、申购、收益和重仓分别记录，不能替代数据日期。
_避免使用_：数据日期、统一更新时间

**业务市场**：
HoldLens 对资产所属公开市场的稳定业务分类，例如 A 股、港股或美股；它不绑定某个第三方行情源的接口编码。
_避免使用_：东方财富市场编码、交易所代码

**交易所代码**：
资产在业务市场内的交易所或板块归属代码，例如 A 股内的上交所、深交所或北交所。
_避免使用_：业务市场、第三方接口市场编号

**数据源市场编码**：
第三方行情源为了请求或解析数据使用的市场编码，例如东方财富 secid 中的市场编号；它只表达数据源适配语义，不作为 HoldLens 的业务市场分类。
_避免使用_：业务市场、交易所代码

**美股**：
HoldLens 的业务市场之一，用于表达美国股票市场公开资产集合；在跨项目刷新契约和 `stock_market` 中建议使用稳定枚举 `US_STOCK`。东方财富 `105`、`106`、`107` 等编号只属于数据源市场编码。
_避免使用_：东方财富市场编码、secid

**股票全量行情刷新**：
针对某个业务市场一次性刷新当前公开股票行情的处理任务。agent 负责抓取并回传结构化结果，server 负责任务状态、幂等、诊断和长期行情事实写入。
_避免使用_：个股补刷、历史行情刷新、用户持仓刷新
