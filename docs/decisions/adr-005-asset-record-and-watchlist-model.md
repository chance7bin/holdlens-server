# ADR-005 以资产记录替代账户持仓模型并拆分自选关系

## 1. 背景

现有 `asset_account`、`asset_info`、`asset_holding` 将账户、用户自选关系、资产身份和当前持仓拆成多张表。该结构要求每条持仓必须归属账户，并让 `asset_info` 同时承担自选关系和持仓标的两种职责；对于以现金、银行卡、基金、股票、债权和负债金额为核心的资产记录场景，账户不是必要事实，且用户无法在“只记基金/股票金额”和“记录具体标的”之间渐进细化。

本决策替代 ADR-001 中关于 `asset_account`、`asset_info`、`asset_holding` 的具体模型建议，并替代 ADR-004 中以 `asset_info` 保存自选关系的存储决策。`portfolio` 仍是用户资产事实源，自选关系仍不代表持仓，公共基金和股票数据仍分别由现有公共数据领域维护。

## 2. 决策

- 删除账户事实和 `asset_info` 中间资产主数据，使用 `asset_record` 统一表示用户的一笔资产、债权或负债金额记录。
- `asset_record` 通过资产目录归类；目录分为不可编辑的系统目录和用户私有目录，并通过 `ADD`、`SUBTRACT` 表达净资产方向。系统初始化使用“投资资产”一级分组，并将基金、股票作为其二级叶子目录；“投资资产”不表达账户事实，也不允许直接记录资产。
- 基金和股票系统目录使用不可变 `catalog_code` 触发专用录入流程。基金、股票记录可以只保存未细分金额，也可以通过 `asset_kind + asset_id` 引用 `fund` 或 `stock_market` 的具体公共标的。
- 未细分金额可以逐笔拆分成具体持仓。拆分生成新记录并扣减源记录，保持同目录、同币种金额守恒；源金额被全部拆完后归档。
- 每条资产记录只以自身 ID 唯一；相同基金或股票可以存在多条记录，汇总时再按 `asset_kind + asset_id` 聚合。
- `asset_record_change` 保存只追加的金额、拆分和生命周期变更历史，不使用历史重放计算当前事实。
- `watchlist_item` 独立保存用户与公共基金/股票的自选关系，不关联资产目录、金额或资产记录。
- `exchange_rate` 保持通用币种对结构，当前只保存“外币作为基准币、人民币作为报价币”的最新汇率；资产记录保存原币金额，汇总查询指定目标币种并经人民币换算。
- 当前数据允许全部重建，不设计旧账户、旧资产和旧变更历史的数据迁移。

## 3. 备选方案

- 保留账户模型：可以表达券商或平台归属，但会把非账户型资产也强制装入账户，并增加录入负担。
- 保留独立资产主表：能够规范化代码和名称，但现有 `fund`、`stock_market` 已拥有公共标的事实，继续保留 `asset_info` 会重复身份并混入自选关系。
- 只保存分类总额：录入最简单，但无法渐进细化到具体基金或股票，也无法支撑后续标的分析。
- 分类总额与具体持仓并存：会产生重复计数；改用“未细分剩余金额 + 具体持仓”后，每笔金额只计算一次。

## 4. 取舍原因

资产记录模型直接匹配用户当前行为：先按目录记录金额，需要时再把基金或股票金额拆成具体标的。删除账户和中间资产主数据后，资产、自选和公共市场数据的事实边界更清楚；代价是 `asset_kind + asset_id` 成为应用层校验的多态引用，数据库不能用单一外键约束两张公共表。

## 5. 影响

- 需要以一次原子变更替换资产相关 DDL、领域模型、持久化映射、查询和开发 Mock 数据。
- 现有自选 API 的外部 `assetRef` 仍可保持不透明契约，但内部关系改为 `asset_kind + asset_id`。
- 基金刷新目标选择需要从 `watchlist_item` 与具体基金持仓中重新取并集；没有公共基金 ID 的未细分金额不产生刷新目标，也不再读取 `asset_info`。
- 公共基金和股票技术主键必须稳定；刷新只能更新或停用现有行，不能删除后重建。
- 当前不包含具体标的分析、数量成本、自动汇率刷新、旧数据迁移或资产跨目录移动。
