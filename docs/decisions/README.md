# 决策记录

这个目录用于记录关键技术或业务方案决策，便于后续回溯。

## 推荐文件命名

- `adr-001-local-data-boundary.md`
- `adr-002-position-calculation-model.md`

## 一篇 ADR 建议包含

1. 背景
2. 决策内容
3. 备选方案
4. 取舍原因
5. 影响范围

## ADR 模板

创建 ADR 时，优先从 [adr-template.md](adr-template.md) 复制模板，并根据决策复杂度补全内容。模板用于统一常见信息结构，不要求所有 ADR 机械保留每个小节；简单决策、低风险取舍或不涉及某类内容时，可以酌情删减、合并或调整小节，但应保留背景、决策内容、取舍原因和影响范围等关键信息。
