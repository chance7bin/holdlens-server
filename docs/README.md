# 项目文档

这个目录用于统一维护项目内文档，尽量避免需求散落在聊天记录、在线文档或临时笔记里。

## 目录说明

- `requirements/`：面向业务和产品的需求文档
- `decisions/`：重要设计决策与取舍记录
- `notes/`：会议纪要、调研记录、临时分析
- `glossary/`：业务术语、领域词汇、统一定义

## 推荐协作方式

1. 新功能先在 `requirements/` 建需求文档。
2. 有重要架构或方案取舍时，在 `decisions/` 记录 ADR。
3. 讨论过程和调研材料统一沉淀到 `notes/`。

## 命名建议

- 目录统一使用英文小写。
- 文件名推荐使用 `主题-日期.md` 或 `编号-主题.md`。
- 需求文档推荐使用 `prd-xxx.md`。
- 决策文档推荐使用 `adr-00x-xxx.md`。
